# Task 2：车牌识别系统方案对比

本项目包含三种基于 **CCPD (Chinese City Parking Dataset)** 数据集实现的车牌识别系统。三个子项目（Member 1, 2, 3）分别探索了不同的技术路线，涵盖了从关键点矫正到端到端 OCR 的多种主流方案。

## 项目概览与导航

|  **子项目**  |        **技术路线**        | **核心特点** |    **常规场景准确率 (All Test)**     |   **鲁棒性场景准确率 (Hard Test)**   |
| :----------: | :------------------------: | :----------: | :----------------------------------: | :----------------------------------: |
| **Member 1** | **YOLOv8-Pose** + **CRNN** | 最佳综合性能 |              **98.70%**              |              **96.44%**              |
| **Member 2** |  **PP-OCRv5** (Det + Rec)  | 最佳识别上限 | 90.04% (端到端)<br />99.30% (仅识别) | 64.87% (端到端)<br />98.21% (仅识别) |
| **Member 3** |  **YOLOv8** + **LPRNet**   |  轻量级基线  |                85.13%                |                59.44%                |

> **注**：准确率指“全字匹配率 (Full Match Accuracy)”。

## 详细方案介绍

### 1. [Member 1: 高精度矫正方案 (YOLOv8-Pose + CRNN)](./task2_member1)

该方案专注于解决车牌在复杂视角下的形变问题，达到了本项目中的**最高端到端精度**。

- **检测模块**：使用 **YOLOv8-Pose** 检测车牌的 4 个角点，通过透视变换（Perspective Transformation）将倾斜车牌“拉正”。
- **识别模块**：使用工业级 **ResNet18 + BiLSTM + CTC (CRNN)** 进行序列识别。
- **亮点**：在倾斜、模糊、阴影等困难场景下（Hard Test）仍保持 96.44% 的极高准确率，推理速度约 70 FPS。

### 2. [Member 2: PP-OCRv5 优化方案](./task2_member2)

该方案基于强大的 **PaddleOCR** 生态进行二次开发，专注于挖掘 OCR 模型的识别潜力。

- **检测模块**：基于 PP-OCRv5 文本检测模型，增加了车牌几何约束和过滤规则。
- **识别模块**：微调 PP-OCRv5_rec 模型，适配“省份+字母+数字”结构。引入了自定义的 `CCPDWarp` 算子处理透视畸变。
- **亮点**：识别模型的单体性能极强（仅识别任务下全对率达 99.30%），且推理速度极快（仅识别 > 150 FPS）。支持图片与视频流的快速部署。

### 3. [Member 3: 轻量级经典方案 (YOLOv8 + LPRNet)](./task2_member3)

该方案实现了一个结构清晰、易于复现的轻量级车牌识别系统。

- **检测模块**：使用标准的 **YOLOv8** 进行目标检测（Bounding Box）。
- **识别模块**：使用改进的 **LPRNet**。利用 CCPD 的 Ground Truth 将 LPRNet 训练为适应“仅框取、未校正”输入的模式。
- **亮点**：系统轻量，结构解耦。在常规场景下表现良好（85.13%），适合作为基础 Baseline 参考。

## 快速使用

每个子文件夹中均包含独立的 `README.md`、环境配置文件（`requirements.txt`）和推理脚本。请点击上方链接进入对应目录查看详细的安装与运行指南。

**通用数据准备**：所有项目均依赖 **CCPD 数据集**（2019+2020）。建议将数据集下载后解压至统一目录，各子项目可通过软链接或配置路径的方式共享数据。
